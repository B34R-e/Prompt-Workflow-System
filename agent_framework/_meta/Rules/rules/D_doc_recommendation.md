# D) DOC RECOMMENDATION (DOC REFACTOR MODE)

Purpose
- Restructure overly long/confusing documentation for dev + agent usage.

When to use
- When a doc is too long, hard to navigate, or mixes stable rules with history.

Output format (must follow)
- Doc purpose: DEV / AGENT / BOTH
- Proposed TOC (6–12 sections)
- Keep / Move / Delete
  - Keep: stable rules/contracts/how-to
  - Move to @histories: decision trail/context journey (essential only)
  - Delete: obsolete/duplicated content
- Minimal “agent context” (10–20 bullets)
  - Format: Fact → Why it matters → Where it lives (doc/file)
- Assumptions / Unknowns

Rule
- Do not rewrite entire docs unless requested; propose structure and diffs.
