# I) ADR_LITE (DECISION LOG)

Purpose
- Capture key technical decisions so the team doesn’t re-litigate them later.

When to use
- Any major design/architecture decision
- Any meaningful trade-off (cost vs speed, correctness vs performance, etc.)
- Any decision that could be questioned later

Output format (must follow)
- Decision (1 line)
- Context (why the decision is needed, 2–4 bullets)
- Options considered (max 3)
- Chosen option + rationale (3–5 bullets)
- Consequences (positive + negative)
- Follow-ups (what must be done next, if any)
- Evidence (@docs/@histories/codebase refs)

Rule
- Keep it short. If it is not a “decision”, do not write an ADR entry.
