# G) DOCS + HISTORIES UPDATE

Purpose
- Keep documentation accurate without bloating it.

Output format (must follow)
- What to update in @docs (agent + dev readability)
- What to record in @histories (essential context journey only)

- Assumptions / Unknowns
- Evidence: which code/config changes triggered these doc updates

Rules
- @docs: stable rules/contracts/how-to
- @histories: decisions/context trail, minimal essentials
- If a major decision was made: ensure ADR_LITE entry exists (module I).
